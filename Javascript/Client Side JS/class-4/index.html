üåê DOM Core: Window and Document Properties

The **Document Object Model (DOM)** is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The document object is your main entry point.

Essential Document Properties

  • window.document: The main object representing the document.
  • document.body: Accesses the <body> element.
  • document.body.innerHTML: Gets or sets the HTML content within the body.
  • document.body.innerText: Gets or sets the visible text content.
  • document.links: A collection (NodeList) of all <a> elements with an href. Access individual links with document.links[i].
  • document.write(): Writes HTML expressions or JavaScript code to a document (use sparingly, as it can overwrite existing content).
  • document.all: A legacy collection of all elements (use selectors instead). Access individual elements with document.all[i].

Document Structure & Metadata

  • document.documentElement: Accesses the root <html> element.
  • document.head.children / document.body.children: HTMLCollection of direct child *elements* (not including text/comment nodes).
  • document.doctype: The document's doctype declaration.
  • document.domain: The domain of the server the document is loaded from.
  • document.URL: The full URL of the document.
  • document.characterSet: The character encoding used (e.g., "UTF-8").
  • document.contentType: The MIME type of the document.
  • document.forms[i]: A collection of all <form> elements. Access properties like .id, .method, .action, .className, and .classList on individual forms.

üéØ Element Selection & Attributes

How to find specific elements in the DOM.

Single Element Selectors

  • document.getElementById('id-name'): Selects the element with the matching **ID**.
  • document.querySelector('selector'): Selects the **first** element that matches the specified CSS selector (e.g., '#id', '.class', 'tagname').

Multiple Element Selectors

  • document.querySelectorAll('selector'): Selects **all** elements that match the specified CSS selector, returning a **NodeList**.

Attributes

  • element.getAttribute('attrName'): Gets the value of a specified attribute (e.g., document.getElementById('id').title is similar to getAttribute('title')).
  • element.setAttribute('attrName', 'newValue'): Sets the value of a specified attribute (e.g., document.getElementById('id').setAttribute('class', 'title')).

Moving between parent, child, and sibling elements/nodes.

DOM Element Relationships (Elements only - ignores text/comment nodes)

  • element.children: Returns an **HTMLCollection** of child **elements**. Access with element.children[i].
  • element.firstElementChild: The first child element.
  • element.parentElement: The closest parent element.
  • element.nextElementSibling: The next sibling element.
  • element.previousElementSibling: The previous sibling element.

DOM Node Relationships (Includes all nodes: elements, text, and comments)

  • element.childNodes: Returns a **NodeList** of all child **nodes**. Access with element.childNodes[i].
  • node.nodeName: The name of the node (e.g., 'DIV', '#text').
  • node.textContent: The text content of the node and its descendants.
  • node.outerHTML: The HTML of the element, including the element itself.
  • node.innerText: The "rendered" text content (takes CSS into account).
  • parentNode **vs** parentElement: parentNode returns any node (element, text, document fragment), while parentElement only returns an element.

üî® Creating, Inserting, Replacing & Removing

Creating New Nodes

  • document.createElement('tagname'): Creates a new element node (e.g., <li>).
  • document.createTextNode('text content'): Creates a new text node.
  • parent.appendChild(child): Adds a node as the **last** child of a parent node.

Inserting Elements

  • parent.insertBefore(newNode, referenceNode): Inserts a newNode before a referenceNode (similar to appendChild but more flexible).
  • targetElement.insertAdjacentElement(position, element)
  • targetElement.insertAdjacentText(position, text)
  • targetElement.insertAdjacentHTML(position, htmlString): This method is highly flexible, supporting four positions relative to the targetElement:
    • 'beforebegin': Before the target element itself.
    • 'afterbegin': Just inside the target element, before its first child.
    • 'beforeend': Just inside the target element, after its last child.
    • 'afterend': After the target element itself.
  • **Custom Challenge:** Implement a custom insertAfter() function using existing methods.

Replacing and Removing

  • parent.replaceChild(newChild, oldChild): Replaces an oldChild node with a newChild node.
  • parent.removeChild(child): Removes a specified child node from the parent.
  • element.remove(): A simpler method to remove the element itself (supported in modern browsers).

üé® Manipulating Styles & Classes

Changing the visual presentation of elements.

Styles

  • element.style.propertyName = 'value': Directly sets an inline CSS style. (e.g., element.style.backgroundColor = 'red').

CSS Classes

  • element.className: Gets or sets the entire class attribute as a string (overwrites existing classes).
  • element.classList: Provides a live list of classes for safer manipulation:
    • element.classList.add('new-class'): Adds one or more classes.
    • element.classList.remove('old-class'): Removes one or more classes.
    • element.classList.toggle('class-name'): Adds a class if it doesn't exist, removes it if it does.
    • element.classList.contains('class-name'): Checks if a class is present (returns boolean).
